Dokumentace k programu: Kalkulátor a validátor kontrolních čísel

1. Popis programu

Tento program jsem vytvořil jako webovou aplikaci v programovacím jazyce Rust za pomoci frameworku Dioxus. Slouží k výpočtu, analýze a validaci kontrolních číslic u běžně používaných identifikačních formátů: Rodné číslo, ISBN-13, EAN-13, IBAN, IČO a čísel platebních karet (Luhnův algoritmus). V aplikaci vizualizuji jednotlivé složky těchto kódů a demonstruji principy modulo aritmetiky, která se k tvorbě kontrolních číslic hojně využívá za účelem zabezpečení dat proti chybám při přenosu či ručním přepisu.

2. Popis funkcí programu

  • Analýza Rodného čísla: Program přijímá prvních 9 číslic rodného čísla. Kontroluje, zda vstup obsahuje pouze číselné znaky, a pomocí algoritmu Modulo 11 počítá 10. kontrolní číslici. V případě validního vstupu rozpadne číslo na logické celky (rok, měsíc, den, koncovka).
  • Analýza ISBN-13: Program zpracovává 12místný vstup oddělený pomlčkami. Verifikuje délku jednotlivých částí (prefix, skupina, vydavatel, publikace) a počítá 13. kontrolní číslici pomocí algoritmu Modulo 10 (střídavé násobení vahami 1 a 3).
  • Analýza EAN-13: Aplikaci jsem navrhl tak, aby parsovala souvislý 12místný číselný řetězec, který dělí na GS1 prefix, kód výrobce a kód produktu. Kontrolní číslici počítá stejným algoritmem (Modulo 10) jako u formátu ISBN. Kód také vizualizuje jako SVG čárový kód pomocí knihovny barcoders.
  • Analýza IBAN: Primárně se pokouším o validaci celého zadaného kódu IBAN pomocí externí knihovny iban. Pokud uživatel zadá platný kód bez kontrolního dvojčíslí, toto dvojčíslí dopočítává aplikace pomocí algoritmu Modulo 97 (po vložení nul a aplikaci matematického vzorce) a následně zkompletovaný IBAN zvaliduje a rozčlení.
  • Analýza IČO (Identifikační číslo osoby): Aplikace přijímá vstup o 7 nebo 8 znacích. V případě 7 znaků vypočítá 8. kontrolní číslici pomocí modula 11 s váhami (8, 7, 6, 5, 4, 3, 2). Pokud je vloženo 8 znaků, validuje, zda zadaná kontrolní číslice odpovídá výpočtu.
  • Luhnův algoritmus (Kreditní karty): Nástroj jsem implementoval pro validaci čísel platebních karet. Pokud je zadáno 15 čísel, program dopočítá 16. kontrolní číslici (postup zprava doleva, násobení každé druhé číslice dvěma, sčítání cifer). Pokud je zadáno 16 čísel, provádí rovnou ověření platnosti.

//Zde algoritmus jak pro EAN-13 tak pro ISBN - aka modulo 10 a nastřídačku *1 a *3
pub fn modulo_10_algorithm(isbn: &str) -> String {
    //parse the string to numbers
    //zde už víme, že máme plné 12 místné isbn, bez kontrolní číslice a zároveň víme, že se skutečně jedná o číslo.
    let mut sum: i32 = 0;
    let mut multiply_by_3: bool = false;

    for s_num in isbn.chars() {
        if multiply_by_3 {
            //zde je 10, protože se pohybujeme v desítkové sustavě
            sum += (s_num.to_digit(10).unwrap() as i32) * 3;
            multiply_by_3 = !multiply_by_3;
        } else {
            sum += s_num.to_digit(10).unwrap() as i32;
            multiply_by_3 = !multiply_by_3;
        }
    }

    //zbytek po vydělení součtu 10 a odečtění tohoto čísla od čisla 10
    let mut res = (10 - (sum % 10)).to_string();
    if res == "10" {
        res = "0".to_string();
    }
    res
    //pokud je výsledek 10, tak se výsledkem stane nula
}

//algoritmus na výpočet kontrolní číslice IČO
pub fn ico_control_digit(ico: &str) -> String {
    //předpokládám, že dostaneme validní IČO číslici (7 čísel - chybí kontrolní číslice)
    //proto zde nebudu zde provádět žádné kontroly.
    let mut total = 0;
    //zpětný for loop - váhy
    //je tam 9, protože indexing
    for (counter, number) in ico.chars().enumerate() {
        //ochrana proti 8 místnému ičo, sice by nemělo být ale kdo ví
        if counter >= 7 {
            break;
        }
        //pokus o castnutí elemenut v 10 soustavě
        if let Some(val) = number.to_digit(10) {
            total += val * (8 - (counter as u32));
        }
    }

    //Výpočet kontrolní číslice ičo
    let control_digit = (11 - (total % 11)) % 10;
    //Vzít číslo a převést na string
    control_digit.to_string()
}

//luhnuv algoritmus pro karty
//celý algoritmus funguje tak, že jede číslem pozpátku, násobí liché indexy 2 a sudé přičítá bez násobení, pokud je vynásobený index větší jak 9, odečte se od něj 9
//poté se provede (10 -(sum%10))%10; a tím vznikne kontrolní číslice
pub fn luhn_algo(number: &str) -> String {
    //počítám s tím, že dostanu 15 místné číslo (číslo karty bez kontrolní číslice)
    let mut sum = 0;
    let mut is_odd = true; // ze základu je odd, protože je odebraná kontrolní číslice a jede se pozpátku

    for num in number.chars().rev() {
        //cast charu na číslo v desítkové soustavě
        if let Some(val) = num.to_digit(10) {
            if is_odd {
                let multiply = val * 2;
                if multiply > 9 {
                    sum += multiply - 9;
                } else {
                    sum += multiply;
                }
            } else {
                sum += val;
            }
            is_odd = !is_odd;
        }
    }

    let res = (10 - (sum % 10)) % 10;
    res.to_string()
}
