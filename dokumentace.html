Dokumentace k programu: Kalkulátor a validátor kontrolních čísel

1. Popis programu

Tento program je webová aplikace vytvořená v programovacím jazyce Rust za pomoci frameworku Dioxus. Slouží k výpočtu, analýze a validaci kontrolních číslic u běžně používaných identifikačních formátů: Rodné číslo, ISBN-13, EAN-13 a IBAN. Aplikace vizualizuje jednotlivé složky těchto kódů a ukazuje principy modulo aritmetiky, která se k tvorbě kontrolních číslic využívá.

2. Popis funkcí programu

  • Analýza Rodného čísla: Program přijímá prvních 9 číslic rodného čísla. Kontroluje, zda vstup obsahuje pouze číselné znaky, a pomocí algoritmu Modulo 11 vypočítá 10. kontrolní číslici.
  • Analýza ISBN-13: Program zpracovává 12místný vstup oddělený pomlčkami. Verifikuje délku jednotlivých částí (prefix, skupina, vydavatel, publikace) a počítá 13. kontrolní číslici pomocí algoritmu Modulo 10 (střídavé násobení vahami 1 a 3).
  • Analýza EAN-13: Aplikace parsuje souvislý 12místný číselný řetězec, který dělí na GS1 prefix, kód výrobce a kód produktu. Kontrolní číslice je počítána stejným algoritmem (Modulo 10) jako u formátu ISBN.
  • Analýza IBAN: Program se primárně pokouší o validaci celého zadaného kódu IBAN pomocí externí knihovny. Pokud uživatel zadá platný kód bez kontrolního dvojčíslí, program toto dvojčíslí dopočítá aplikací algoritmu Modulo 97 a následně zkompletovaný IBAN zvaliduje.

3. Zdrojový text programu a vysvětlivky

Zdrojový kód je napsán v jazyce Rust. Pro ošetření chyb (např. při parsování textu na čísla) využívá typy Option a Result, které zajišťují bezpečný běh aplikace i při zadání nestandardních vstupů.

3.2. Soubor: algorithms.rs

Vysvětlivka: Soubor obsahuje sdílené matematické funkce nezávislé na uživatelském rozhraní. calculate_rc_control_digit ověřuje, zda lze vstup parsovat na celé číslo (typ i64) a následně provádí operaci modulo 11. Funkce modulo_10_algorithm iteruje přes jednotlivé znaky řetězce (ISBN/EAN), převádí je na čísla a na základě střídavého příznaku multiply_by_3 provádí výpočet váženého součtu. 
pub fn calculate_rc_control_digit(rodne_cislo: &str) -> Option<i32> {
    if let Ok(val) = rodne_cislo[..9].parse() {
        let rodc_int: i64 = val;
        let mut modulo_rodc: i32 = (rodc_int % 11) as i32;
        if modulo_rodc == 10{
        	modulo_rodc = 0;
        }
        return Some(modulo_rodc);
    }

    None
}

pub fn modulo_10_algorithm(isbn: &str) -> String{
	let mut sum: i32 = 0;
	let mut multiply_by_3: bool = false;
	
	for s_num in isbn.chars(){
		if multiply_by_3{
			sum += (s_num.to_digit(10).unwrap() as i32) * 3;
			multiply_by_3 = !multiply_by_3;
		}
		else{
			sum += s_num.to_digit(10).unwrap() as i32;
			multiply_by_3 = !multiply_by_3;
		}
	}

	let mut res = (10-(sum%10)).to_string();
	if res == "10"{
		res = "0".to_string();
	}
	res
}

3.4. Soubor: isbn.rs

Vysvětlivka: Soubor zpracovává formát ISBN. Namísto indexace fixních pozic znaků využívá metodu split('-'), která uživatelský vstup rozdělí podle vložených pomlček do takzvaného vektoru (pole prvků). U každého extrahovaného prvku se kontroluje správný formát (číslice) i přípustná délka pro danou sekci (např. max 5 znaků pro skupinu). Pokud součet délek dává 12 znaků, je zavolán výpočetní algoritmus.

	if !input_value.is_empty(){
		let input_string = input_value();
		let mut fields_vec: Vec<&str> = input_string.split('-').collect();
		fields_vec.retain(|&elem| !elem.is_empty());

		if fields_vec.len() >= 1 {		
				if fields_vec[0].parse::<i32>().is_err(){
					has_error = true;
					error_text = "V prefixu ISBN se vyskytuje nečíselný charakter.".to_string();
				}

					isbn_prefix = fields_vec[0].to_string();

					}
